## Your First Component

Creating a Web Component with Pencel requires no configurationâ€”just decorators and TypeScript.

### Define the Component

Create a new file `components/Hello.tsx` with a simple component:

```tsx
import { Component, Prop } from '@pencel/runtime';

@Component({
  tag: 'hello'
})
export class Hello extends HTMLElement {
  @Prop() name: string = 'World';

  render() {
    return <span>Hello {this.name}</span>;
  }
}
```

The `@Component` decorator registers your class as a custom element with the default `pen` namespace. The resulting tag is `pen-hello`.

<Aside type="tip">
**Tag naming:** Pencel automatically prefixes your component tag with the namespace (default: `pen`), so `tag: 'hello'` becomes `pen-hello`. This ensures no conflicts with browser or third-party components.
</Aside>

### Use the Component

Add it to your HTML file:

```html
<pen-hello name="Max"></pen-hello>
```

Or in JSX/TSX:

```tsx
<pen-hello name="Max" />
```

## Multi-Framework Support

By default, Pencel outputs native Web Components. To generate framework-specific bindings, enable output plugins in your configuration:

```ts
// pencel.config.ts
import { defineConfig } from '@pencel/cli';

export default defineConfig({
  outputs: {
    react: { path: 'out/react' },
    angular: { path: 'out/angular' }
  }
});
```

<Aside type="note">
Output plugins are configured in `pencel.config.ts`. See [Framework Outputs](/pencel/outputs) for details on available options.
</Aside>
